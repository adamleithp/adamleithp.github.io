Recently I've gotten reaaaal tired of jQuery. The once go-to tool for DOM manipulation has crippled my vanilla javascript abilities, and I say no more. In this post I will be walking through a CSS-only dropdown, that has all the recommended tags and functionality required for a highly functional website.

It's taken me a while to fully understand the importance of useabilty in my code. Tabbing, aria roles, alt tags and title tags, are all very crucial to your websites functionality, specifically to people with disabilites. These aformentioned traits may not seem crucial to us 21"-iMac-mouse-event-driven-browsers' but play a very real role to those who have a disability (ie. Blindness, Deaf and Lack of motorskills in the limbs).

I like to play a game (really just QA); the game is to get to every page, and every link by using nothing but tab and enter. If your main navigation is hidden inside a dropdown, you will quickly realize how this can be a bother.

The Menu, by pressing a key

HTML
	
<button tabIndex="0" 
    aria-expanded="false" 
    aria-controls="collapsible-0">
    Menu
</button>

<nav aria-hidden="true">
    <ul class="menu-list">
        <li>
            <a href="#">Menu Item #1</a>
        </li>
        <li>
            <a href="#">Menu Item #2</a>
        </li>
    </ul>
</nav>

<a href="#" 
    role="link">
    Next Tab Index
</a>
CSS
	
header {
    position: relative;
    background:#f3f3f3;
}
a, button {
    &:focus {
        outline: 1px dotted blue !important;
    }
}
[aria-hidden="true"] {
    display:none;
}    

[aria-hidden="false"] {
    display:block;
}
jQuery (yes I know)
	
$('button').each(function() {
  
  var $this = $(this),
      nav = $('nav'),
      button = $('button');

  button.on('click', function() {
    var state = $(this).attr('aria-expanded') === 'false' ? true : false;
    $(this).attr('aria-expanded', state);
    nav.attr('aria-hidden', !state);
  });
});
